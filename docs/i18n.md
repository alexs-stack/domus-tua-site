# i18n — Multilingua Domus Tua

Il sito Domus Tua è multilingua. Le traduzioni sono **complete e verificate (QA)**
sulla homepage e su tutte le sottopagine, quindi l'i18n resta attivo di default —
ma è **controllabile tramite feature flag** in caso serva tornare temporaneamente
al solo italiano (es. deploy MVP prima di una revisione legale, vedi sotto).

---

## Lingue supportate

| Codice | Lingua     | Sigla |
| ------ | ---------- | ----- |
| `it`   | Italiano   | IT    |
| `en`   | English    | EN    |
| `fr`   | Français   | FR    |
| `de`   | Deutsch    | DE    |
| `es`   | Español    | ES    |

- **Lingua principale:** italiano (`defaultLocale = "it"`).
- L'italiano è anche il fallback: se il cookie è assente o non valido si parte da `it`.

---

## Architettura

L'i18n è interamente **lato client**, senza routing per-lingua (niente `/en`, `/fr`
nell'URL): la lingua è una **preferenza utente salvata in cookie**, non una rotta.
Questo mantiene un solo set di URL canonici e preserva la SEO locale italiana.

### 1. Provider + cookie — `app/components/i18n/LocaleProvider.tsx`

- Espone il context con `locale`, `setLocale(l)` e il dizionario corrente `d`.
- La preferenza è salvata nel cookie **`dt_locale`** (`max-age` 1 anno, `path=/`,
  `samesite=lax`).
- **SSR-safe:** il server rende sempre `defaultLocale` (`it`); al primo mount il
  client legge il cookie e si allinea, aggiornando anche `document.documentElement.lang`.
  Questo evita il mismatch di hydration.
- Hook disponibili:
  - `useLocale()` → `{ locale, setLocale, d }`
  - `useDict()` → scorciatoia per il dizionario corrente (`useLocale().d`).

### 2. Selettore lingua — `app/components/i18n/LanguageSwitcher.tsx`

- Dropdown accessibile (`role="listbox"` / `role="option"`, `aria-*`) con l'icona
  Globe; supporta la variante `light` per header su sfondo scuro.
- **Gated dietro feature flag** (vedi sezione successiva).

### 3. Due pattern di traduzione coesistenti

**a) Dizionario centrale** — `app/lib/i18n/dictionaries.ts`

Per chrome ricorrente e strutture stabili (nav, header, hero, ricerca, footer, ecc.).
Definisce `locales`, `defaultLocale`, `localeNames`, `localeShort`, il tipo `Dict`
e l'oggetto `dictionaries` con le 5 lingue.

```ts
import { useDict } from "./i18n/LocaleProvider";
const d = useDict();
// d.hero.title1, d.nav.vendi, ...
```

**b) Dizionario inline per-componente**

Per copy specifico di un singolo componente/sezione. Il dizionario vive nel file
stesso, con tutte e 5 le lingue:

```ts
import { useLocale } from "./i18n/LocaleProvider";
const copy = { it: {...}, en: {...}, fr: {...}, de: {...}, es: {...} };
const { locale } = useLocale();
const c = copy[locale];
```

> Quando si aggiungono nuove stringhe visibili a un componente con dizionario inline,
> vanno inserite in **tutte e 5 le lingue** con traduzioni naturali e premium
> (apostrofi tipografici ’, accenti corretti). Nomi propri **non** tradotti:
> Domus Tua, Open Domus, Domus D.O.C., Domus di Origine Certificata, RealSmart,
> Raffaela Rizza, Tradate, Varese, Google, Trustindex, WhatsApp, YouTube, Instagram.

### Copertura

Homepage completa e tutte le sottopagine (vendi, acquista, metodo, open-domus, case
+ dettaglio immobile, recensioni, chi-siamo, contatti, privacy, cookie) più tutto il
chrome (header, footer, WhatsApp float). ~35 componenti/pagine usano gli hook i18n.

## Abilitare / disabilitare l'i18n

Il selettore lingua è controllato dalla variabile d'ambiente:

```
NEXT_PUBLIC_ENABLE_I18N
```

| Valore                    | Comportamento                                                        |
| ------------------------- | -------------------------------------------------------------------- |
| assente (non impostata)   | **i18n attivo** — selettore visibile (default)                       |
| `"true"`                  | **i18n attivo** — selettore visibile                                 |
| `"false"`                 | **i18n disattivato** — `LanguageSwitcher` rende `null` (solo IT)     |

Note tecniche:

- La variabile ha prefisso `NEXT_PUBLIC_` perché viene letta **lato client** e
  inlined a build-time da Next.js. **Un cambio di flag richiede una nuova build**
  (non è un toggle runtime).
- Con il flag a `"false"` viene nascosto **solo il selettore**: i dizionari e il
  `LocaleProvider` restano in bundle, quindi il sito parte e resta in italiano
  (`defaultLocale`). Chi ha già un cookie `dt_locale` valido vedrebbe comunque la
  propria lingua salvata; per un vero solo-IT si può inoltre azzerare il cookie o
  ignorarlo in fase di deploy.
- Il gate è applicato **dopo** l'esecuzione di tutti gli hook (`useLocale`,
  `useState`, `useRef`, `useEffect`): l'ordine degli hook resta invariato e le
  regole di React sono rispettate.

Esempio (`.env.local` o variabili d'ambiente dell'hosting):

```
# Sito solo italiano (selettore lingua nascosto)
NEXT_PUBLIC_ENABLE_I18N=false
```

## Checklist QA (prima del go-live)

- [ ] Cambio lingua dal selettore aggiorna **tutte** le sezioni visibili (niente
      stringhe rimaste in italiano quando si sceglie EN/FR/DE/ES).
- [ ] La scelta persiste dopo un refresh e tra le pagine (cookie `dt_locale`).
- [ ] Nessun **mismatch di hydration** in console al primo caricamento.
- [ ] `document.documentElement.lang` riflette la lingua scelta.
- [ ] Homepage **e** ogni sottopagina verificate nelle 5 lingue (vendi, acquista,
      metodo, open-domus, case + dettaglio, recensioni, chi-siamo, contatti,
      privacy, cookie).
- [ ] Nomi propri non tradotti (Domus Tua, Open Domus, Domus D.O.C., RealSmart,
      Raffaela Rizza, Tradate, Varese, ecc.).
- [ ] Tipografia corretta: apostrofi ’, accenti, niente troncamenti che rompono il
      layout premium.
- [ ] Selettore accessibile da tastiera; `aria-label`/`aria-selected` corretti;
      focus visibile.
- [ ] Con `NEXT_PUBLIC_ENABLE_I18N=false` il selettore **non** compare e il sito
      resta interamente in italiano.
- [ ] Testo legale (privacy/cookie): vedi nota sotto.

## Note importanti

### La metadata resta in italiano (SEO locale)

I metadati (title, description, Open Graph, ecc.) **restano in italiano** anche con
l'i18n attivo. Domus Tua è un'agenzia locale (Tradate, prov. Varese) e la SEO è
mirata al mercato italiano: mantenere metadata e URL canonici in italiano protegge
il posizionamento locale. La traduzione riguarda l'**esperienza on-page**, non la
struttura SEO.

### Testo legale tradotto a macchina → revisione professionale prima del go-live

Le pagine **privacy** e **cookie** contengono testo legale. Le traduzioni non
italiane di questo testo sono, ad oggi, **traduzioni automatiche/di lavoro** e
**devono essere riviste da un professionista** (traduttore legale / consulente
privacy) prima della messa in produzione con l'i18n attivo. Un testo legale
tradotto in modo impreciso è un rischio di conformità.

### MVP: l'italiano-only è la scelta più sicura se le traduzioni non sono riviste

Come regola generale, per un MVP **il solo italiano è l'opzione più sicura** finché
le traduzioni non sono state verificate: si evita di pubblicare copy o testo legale
non revisionato. **In questo caso**, però, le traduzioni delle pagine sono già state
sottoposte a QA su homepage e sottopagine, quindi l'i18n resta attivo — con l'unica
riserva del testo legale sopra. Il flag `NEXT_PUBLIC_ENABLE_I18N=false` resta come
via di sicurezza per tornare al solo italiano senza rimuovere codice.
